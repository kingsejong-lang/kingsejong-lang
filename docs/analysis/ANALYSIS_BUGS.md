# 버그 추적 (Phase 8 기준)

**기준 버전**: v0.8.0-dev  
**최종 업데이트**: 2025-11-20  
**테스트 상태**: 1,490/1,490 통과

현재 메인 브랜치에서는 공개된 재현 가능한 버그가 없습니다. 과거 보고서에서 언급되던 SEGFAULT, unsafe dynamic_cast, 주석 파싱 문제 등은 모두 해결되었고, 관련 문서도 최신 상태로 정리되었습니다.

## 모니터링 항목
| 영역 | 감시 내용 | 메모 |
|------|-----------|------|
| 회귀 테스트 | regression_test.py (성능) | Phase 8.4에서 CI 통합 필요 |
| 튜토리얼 코드 | 타입 선언/`타입()` 함수 사용 | 사용자 문서 갱신 완료 후 재검증 예정 |
| stdlib/http & db | 신규 모듈 작성 계획 | 구현 전까지 테스트 공백 존재 |

## 대응 프로세스
1. 회귀 테스트 실패 시 우선순위 P0로 분류.
2. 문서/튜토리얼에서 발견된 코드 불일치는 bug label 대신 docs label로 처리.
3. Phase 9 이후에는 Incremental GC, 메모리 풀링 등 신규 기능이 추가되므로 MemorySafetyTest 확장 계획을 수립한다.
